# Dashboard Implementation Plan

## Goal

Build the dashboard as a showcase for one complete run through the data-generation pipeline.

The dashboard shows a single pre-existing run. The run contains one source video with its full data-generation pipeline:

```text
source video
  -> object detection
  -> tracking overlays
  -> annotation timeline
  -> world-model data
  -> Seedance synthetic outputs
```

## Scope — Demo Build

This is a demo-scoped build. Show one run end to end with real data, not a product with full CRUD.

### In Scope

- One seeded run with real pipeline data.
- Source video player with synced bbox overlays.
- Action phase timeline.
- Data inspector: objects, tracks, relations, world model, raw JSON.
- Synthetic output gallery with generated video cards.
- Modern dark UI that looks polished.
- Static seed data from the e2e pipeline output.

### Out of Scope (Future)

- Run library with search and filters.
- Multi-source-video support.
- Upload, reprocess, and export flows.
- Backend API server.
- Authentication.

## Build Strategy

Build in vertical slices. Each slice delivers a working feature, not a shell.

All data comes from static JSON seed files generated by the e2e pipeline script.

## Phase 1 — Foundation and Layout

### Build

- React + Vite + Tailwind project.
- Dark theme shell.
- Run header with metadata.
- Seed data loader.

### Done When

- `/dashboard` renders a dark-themed page.
- Run name, status, target object, and action label are visible.
- Seed data loads from static JSON.

## Phase 2 — Video Player and Overlays

### Build

- HTML5 video player component.
- Canvas overlay layer.
- Timestamp synchronization.
- Bbox rendering for tracked objects.
- Labels with confidence badges.
- Track visibility toggles.

### Done When

- Source video plays with synced bbox overlays.
- Scrubbing updates overlays.
- Pausing freezes overlays.
- Hand and object tracks are visible with stable colors.

## Phase 3 — Timeline and Data Inspector

### Build

- Action phase timeline bar.
- Phase labels and click-to-jump.
- Data inspector with tabs: Objects, Actions, Relations, World Model, Raw JSON.
- Current-timestamp data binding.

### Done When

- Timeline shows colored action phases synced to video.
- Clicking a phase jumps video to that time.
- Inspector shows current-frame objects and relations.
- Raw JSON is viewable.

## Phase 4 — Synthetic Output Gallery

### Build

- Synthetic output cards.
- Video preview per card.
- Prompt viewer.
- Source-to-synthetic comparison layout.
- Status badges.

### Done When

- Generated videos display in a gallery.
- Each card shows label, status, prompt, and video.
- User can compare source and generated video.

## E2E Pipeline

A Python script that takes a source video and produces all seed data:

1. Extract frames at intervals.
2. Run object detection via BytePlus Vision API (`seed-2-0-pro-260328`).
3. Build object tracks from per-frame detections.
4. Generate action labels via BytePlus LLM (`seed-2-0-lite-260228`).
5. Build world model from tracks and actions.
6. Submit Seedance generation jobs (`seedance-1-5-pro-251215`).
7. Poll for completion and download generated videos.
8. Output all data as JSON seed files for the dashboard.

This pipeline produces real data that the dashboard renders.
